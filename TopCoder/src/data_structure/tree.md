### 이진 트리 자료구조
- 트리는 계층적 관계를 표현하는 자료구조(조직도 등)
-용어
	- 노드 : 트리의 각 구성요소
	- 간선(edge) : 노드와 노드를 연결하는 연결선
	- 루트노드(root node) : 트리 구조에서 최상위에 존재하는 노드
	- 단말노드(terminal node) : 아래로 또 다른 노드가 연결되어 있지 않은 노드
	- 내부노드(internal node) : 단말 노드를 제외한 모든 노드
- 정의
	- 루트 노드를 중심으로 두 개의 서브 트리로 나뉘어 있는 트리
	- 각 서브 노드 또한 두 개의 트리로 나뉘어 있는 이진 트리
	- 즉, 재귀적 용어. 
	- 노드가 존재하지 않는 곳은 공집합의 노드가 존재한다고 파악함
- 포화 이진 트리와 완전 이진 트리
	- 용어
		- 레벨 : 각 층별로 숫자를 매긴 것
		- 높이 : 트리의 최고 레벨
	- 포화 이진 트리 : 레벨과 높이가 일치하는 트리이면서 모든 노드가 공집합이 아닌 노드로 구성된 트리
	- 완전 이진 트리 : 레벨이 높이에 일치하여 완전하게 공집합이 아닌 노드로 채워져 있지는 않지만 각 레벨은 공집합이 아닌 노드로 완전하게 채워져 있는 트리
	
- 구현
	- 배열기반
		- 탐색이 매우 용이하다
		- 각 노드에 번호가 부여되어 있어 해당 번호를 인덱스로 사용할 수 있는 경우에 제한적으로 사용가능
		- 번호 중 0번은 사용하지 않음. 절대적이지는 않지만 편의성이 높고 실수할 위험이 줄어든다.
		- 완전 이진트리의 구조를 갖는 힙(heap)은 이를 기반으로 구현되었다.
		- 트리의 구조를 한 눈에 알아보기 어려우며, 동적으로 크기를 변경할 수 없다는 단점이 있다.
	- 연결리스트기반
		- 트리 구조를 구현하기에 더 적합
	- ADT
		- SetData(Object data) : 노드에 데이터 저장
		- getData() : 데이터 반환
		- makeLeftSubTree(Node parent, Node child) : 왼쪽 서브트리를 연결
		- makeRightSubTree(Node parent, Node child) : 오른쪽 서브트리를 연결
		- getLeftSubTree() : 왼쪽 서브트리에 저장된 데이터 반환
		- getRightSubTree() : 오른쪽 서브트리에 저장된 데이터 반환
- 순회(재귀적으로 해결)
	- 중위순회 : 왼쪽 자식 - 부모 - 오른쪽 자식 노드 순서로 순회
	- 전위순회 : 부모 - 왼쪽자식 - 오른쪽 자식 노드 순서로 순회
	- 후위순회 : 왼쪽자식 - 오른쪽 자식 - 부모 노드 순서로 순회

### 수식트리(Expression Tree)
- 이진 트리를 이용해 수식을 표현하는 것.
- 연산의 순서가 명확하며, 연산의 과정을 쉽게 파악할 수 있는 이진 트리의 일종
- 그러나 중위 표기법의 상태인 수식을 바로 이진 트리로 표현하는 것은 어려움. 따라서, 후위 표기법으로 바꾸어서 해당 수식을 수식 트리로 변경하는 것이 좋음.
