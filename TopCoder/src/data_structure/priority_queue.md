### 우선순위큐
- 먼저 들어간 데이터의 여부 즉, 순서에 상관없이 우선순위가 높은 데이터가 먼저 나온다.
- 각각의 데이터가 우선순위를 지닌다기 보다, 데이터를 근거로 우선순위를 판단가능해야한다.
- 구현은 배열, 연결 리스트, 힙을 이용한 3가지 방법이 있다.
- 구현
	- 배열
		- 데이터를 우선순위가 높을 수록 배열의 앞쪽에 데이터를 위치시킴
		- 이 경우 삽입 및 삭제시마다 데이터를 한 칸씩 뒤 혹은 앞으로 이동시켜야 한다.
		- 삽입의 위치를 찾기 위해 배열에 저장된 모든 데이터와 우선순위를 비교해야 할수도 있다.
	- 연결리스트
		- 배열처럼 매 순간 데이터를 한 칸씩 이동시킬 필요는 없다.
		- 그러나 삽입 위치를 찾기 위해 비교해야 한다는 빈번하게 일어나는 단점은 극복 못함
	- 힙
		- 힙은 이진 트리이되 완전한 이진트리를 의미한다.
		- 또한 모든 노드에 저장된 값은 자식 노드에 저장된 값보다 크거나 같아야 한다. 
		- 즉, 루트 노드의 값이 가장 커야만 한다.
		- 루트 노드로 올라갈수록 값이 커지는 완전 이진 트리가 최대 힙이며, 루트노드로 올라갈수록 값이 작아지면 최소 힙이라고 한다.
- 힙의 구현
	- 데이터 저장
		- 새로운 데이터는 우선순위가 가장 낮다는 가정하에서 '마지막 위치'에 저장한다.
		- 부모 노드와 우선순위를 비교하여 위치가 바뀌어야 한다면 바꿔준다. 바뀐 다음에도 계속해서 부모 노드와 비교한다. 위치를 찾을 때까지 반복
	- 데이터 삭제
		- 마지막 노드를 루트 노드의 자리로 옮기고 자식 노드와의 비교를 통해 제자리를 찾도록 한다.
	- 힙의 시간, 공간복잡도 : O(log2n)
	- 기반
		- 힙은 배열로 구현한다.
		- 연결리스트를 기반으로 힙을 구현시, 새로운 노드를 힙의 마지막 위치에 추가하기가 쉽지 않다.
		- 따라서, 노드에 고유한 번호를 부여하여 각 번호가 배열의 인덱스가 되도록 구현한다.(인덱스 0 은 사용하지 않는다.)
		- 왼쪽 자식 노드의 인덱스 : 부모 노드의 인덱스 * 2
		- 오른쪽 자식 노드의 인덱스 : 부모 노드의 인덱스 * 2 + 1
		- 부모 노드의 인덱스 : 자식 노드의 인덱스 / 2
	- 특징
		- 힙은 완전 이진 트리
		- 힙의 구현은 배열을 기반으로 인덱스가 0인 경우는 비워둠
		- 힙에 저장된 노드의 개수와 마지막 노드의 고유번호는 일치한다.
		- 노드의 고유번호가 노드가 저장되는 배열의 인덱스 값
